# Visual Identity

## Visual Direction

Warm & approachable

- The system should feel like a thoughtful letter, not a dashboard.
- It should feel calm, personal, and clear on first read.
- It should help people face the numbers without feeling judged.
- It should look more like Marcus's practice than a generic finance app.

## Design Intent

- Lead with calm, not urgency.
- Use warmth to make financial reality easier to face.
- Keep layouts simple and spacious so the content feels considered.
- Let hierarchy come from type, contrast, and rhythm rather than heavy UI chrome.
- Make interactive moments feel guided and human, not mechanical.

## Colors

| Role | Hex | Usage |
|------|-----|-------|
| Primary | `#234736` | Primary anchor for headings, buttons, key dividers, and structural elements |
| Secondary | `#A79E92` | Muted neutral for borders, secondary labels, dividers, and quiet surfaces |
| Accent | `#C46F52` | Warm highlight for active progress, selected states, and measured emphasis |
| Background | `#F6F1E8` | Warm paper-toned page background and large surfaces |
| Surface | `#FFFDFC` | Main content cards, form panels, and letter-style containers |
| Text | `#1F2420` | Primary body copy, headings, and labels with strong contrast |
| Text Secondary | `#5F655E` | Explanatory copy, helper text, and subdued metadata |

## Color Usage Rules

- Background should stay warm and light. Avoid stark white unless needed for input fields.
- Primary green carries trust and structure. Use it more than the accent.
- Accent is for sparing emphasis. It should guide the eye, not dominate the page.
- Secondary neutral should keep the interface grounded when green would feel too heavy.
- Success should stay calm. Use primary green with supportive copy rather than introducing a brighter success color.
- Progress should use the accent when something is active or moving forward.
- Attention states should use the accent plus plain language when something needs care or follow-up.
- Do not use the accent for generic decoration if it also signals attention or progress elsewhere.
- Avoid alarm-red semantics. Hard news should feel steady, not punitive.

## Typography

| Role | Font |
|------|------|
| Heading | `Cormorant Garamond`, `Georgia`, `Times New Roman`, serif |
| Body | `Source Sans 3`, `Segoe UI`, `Helvetica Neue`, Arial, sans-serif |

## Type Guidance

- Headings should feel personal and editorial. Use the serif with enough size and line height to echo a letter, not a brochure.
- Body text should stay calm and highly readable. Favor regular and semibold weights over many type variations.
- Avoid all caps for body UI. It reads colder and more corporate.
- Keep number styling straightforward and legible. Do not make key figures feel flashy.
- Fallback fonts should preserve a calm reading rhythm if the preferred web fonts are unavailable.

## Accessibility Expectation

- Body text and key interface text should meet at least WCAG AA contrast against their background.
- Do not rely on color alone to signal progress, success, or attention. Pair color with words or icons.
- Helper text, labels, and status messaging should remain readable on warm backgrounds, not fade into the page.

## Layout Guidance

- Favor a single-column reading rhythm on core decision screens.
- Group content into clear sections with generous spacing, like paragraphs in a letter.
- Keep cards soft and quiet: light borders, minimal shadows, comfortable padding.
- Use horizontal rules, spacing, and headings to create structure instead of boxed-in dashboards.
- Mobile layouts should preserve the same calm pacing rather than compressing everything into dense stacks.

## Component Direction

- Inputs: simple, clearly labeled, with visible focus states in the primary green.
- Buttons: solid primary green for the main next step, subtle ghost or outlined treatments for secondary actions.
- Progress indicators: thin bars or step markers using the accent for active progress, with text that explains what comes next.
- Result panels: headline first, short interpretation second, next-step guidance third.
- Empty space: intentional and protected. Do not fill the interface with decorative widgets.

## Things To Avoid

- High-saturation blues, bright greens, or fintech gradients.
- Dense tables as the main emotional entry point.
- Heavy card shadows, glossy effects, or calculator-style control clusters.
- Visual language that suggests scoring, ranking, or automated judgment.
